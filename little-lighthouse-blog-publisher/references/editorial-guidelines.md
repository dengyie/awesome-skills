# Editorial Guidelines

Use this reference while collecting body and metadata.

## Body Readiness

Classify the article as:

- `ready`: coherent intro, sections, and ending
- `needs-light-edit`: publishable after small title, excerpt, or heading fixes
- `draft-only`: notes, outline, or incomplete article
- `blocked`: not enough content to create a meaningful post

If readiness is `draft-only`, recommend draft publication, request missing sections, or propose an outline expansion.

## Suggestions To Provide

Review:

- title clarity
- opening clarity
- section structure
- missing conclusion
- code block formatting
- useful heading hierarchy for a TOC
- whether the article should be draft or published

Do not rewrite the whole article without user consent.

## Metadata Quality

- `excerpt` should state the article's value, not repeat the title.
- `seoDescription` can match `excerpt` when concise.
- `summaryQuote` should be omitted if it sounds forced.
- `tags` should usually contain 2-5 specific discovery terms.
- `featured` defaults to `false` unless explicitly requested.
- `date` defaults to current local date only after publication timing is confirmed.

## Chinese Content

For Chinese posts:

- suggest 2-3 readable English slug candidates
- use Chinese excerpt unless the user asks for bilingual metadata
- normalize titles only when clarity improves
- keep technical nouns stable rather than translating word by word

## Code Blocks

Check that:

- fenced language labels exist where useful
- code blocks are not accidental indented prose
- commands and output are separated
- long lines are acceptable for the article style
