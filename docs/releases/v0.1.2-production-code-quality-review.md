# v0.1.2: Installation And Packaging Polish

## Summary

This release tightens the last productization edges around local installation and GitHub-facing packaging.

It focuses on:

- aligning helper scripts with the current `~/.agents/skills` installation path
- preserving compatibility with legacy `~/.codex/skills` setups
- improving README discoverability for GitHub users
- removing tracked cache artifacts from the shipped skill

## What Changed

### Installation

`install-local-skill.sh` and `update-local-skill.sh` now install to `~/.agents/skills` by default.

They also support legacy local Codex copies:

- keep `~/.codex/skills` present, or
- set `INSTALL_LEGACY_CODEX_COPY=1`

and the scripts will sync both locations.

### Documentation

The repository landing page now documents:

- helper-based installation
- compact review brief mode
- repo layout for faster navigation

The usage docs now explain the helper script behavior directly.

### Packaging

Removed tracked `__pycache__` artifacts from the skill package so the repository ships cleaner source-only contents.

## Validation

Validated with:

```bash
python3 -m unittest discover production-code-quality-review/tests -v
bash production-code-quality-review/scripts/install-local-skill.sh
bash production-code-quality-review/scripts/update-local-skill.sh
python3 production-code-quality-review/scripts/review-entrypoint.py --repo . --format compact
```

## Notes

This is a polish release. The review model, deterministic context collection, and reference-routing behavior stay unchanged.
