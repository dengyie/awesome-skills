# v0.1.6: Skill Package Contract

## Summary

This release strengthens the `production-code-quality-review` skill package contract without adding repository sprawl.

It makes the skill directory easier to recognize as the protected core asset, adds lightweight schemas for downstream tooling, and improves deterministic review context collection across branch, tracked, and untracked changes.

## What Changed

### Core Skill Asset

- Added a package-level README inside `production-code-quality-review/`.
- Made `production-code-quality-review/` more explicit as the project core in the root README.
- Kept development notes isolated under `docs/dev/`.

### Schema Contracts

- Added `review-context.schema.json` for generated review context.
- Added `finding.schema.json` for machine-readable review findings.
- Added regression coverage for the schema contract and required core package files.

### Review Context Collection

- `working_tree` scope now combines branch diff, tracked worktree diff, and untracked text file ranges.
- Base inference now considers remote default branch, `develop`, and `trunk`.
- Changed-line coverage is more complete for local review workflows.

### Verification Suggestions

- Improved package-manager-aware suggestions for `pnpm`, `yarn`, `bun`, and npm projects.
- Added detection for Python checks such as `pytest`, `ruff`, and `mypy`.
- Avoided suggesting guaranteed-missing JavaScript commands when `package.json` has empty scripts.
- Bun projects with test scripts now suggest `bun run test`.
