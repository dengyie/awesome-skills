# v0.1.3: Installed Copy Update Recovery

## Summary

This release fixes the highest-priority maintenance bug in the local skill workflow.

It focuses on:

- making installed skill copies remember their source checkout
- allowing `update-local-skill.sh` to refresh safely when invoked from the installed path
- documenting the safer update behavior

## What Changed

### Update Flow

Installed copies now persist their source checkout path in `.skill-source-dir`.

`update-local-skill.sh` reads that metadata and refreshes from the recorded source checkout, so an installed copy can update itself safely without deleting its own source tree.

### Reliability

Added regression coverage for the installed-copy update path.

### Documentation

Updated README, quickstart, golden path, changelog, and release docs to reflect the recorded-source refresh behavior.

## Validation

Validated with:

```bash
python3 -m unittest discover production-code-quality-review/tests -v
bash production-code-quality-review/scripts/install-local-skill.sh
bash /Users/mango/.agents/skills/production-code-quality-review/scripts/update-local-skill.sh
```

## Notes

This is a focused patch release on top of `v0.1.2`. The review model and helper surface stay the same; the maintenance path is now safer and more predictable.
