# v0.1.0: Production Code Quality Review

## Summary

This release turns `production-code-quality-review` from a small review SOP into a more complete, testable Codex skill package.

The skill now combines:

- a clear review workflow
- deterministic repo-context collection
- changed-line mapping
- stack-aware reference routing
- explicit false-positive control
- a structured review output contract

## Why This Release Exists

The original version had a strong review philosophy, but it still behaved more like a high-quality prompt than a reusable skill product.

Main gaps addressed in this release:

- unstable review scope collection
- thin stack-specific guidance
- no deterministic changed-line analysis
- no machine-readable context output
- no tested helper tooling
- no formal output contract
- too much structural sprawl for long-term maintenance

## Highlights

### 1. Deterministic review context collection

Added a Python helper layer that can inspect a git repository and emit structured review context.

Key capabilities:

- infer base branch or fallback scope
- separate staged, unstaged, and untracked files
- collect changed files
- parse `git diff --unified=0` into changed-line ranges
- detect stack signals from file patterns
- derive risk flags
- suggest safe verification commands

Main files:

- `production-code-quality-review/scripts/review_skill_lib.py`
- `production-code-quality-review/scripts/collect-review-context.py`
- `production-code-quality-review/scripts/review-entrypoint.py`

### 2. Human-readable review brief output

Added a unified review entrypoint that can generate:

- machine-readable JSON
- markdown review briefs

This makes the skill easier to use both from Codex and from local automation or terminal workflows.

### 3. Better review routing

The skill now routes more intentionally:

- always load core review controls first
- add stack-aware references only when relevant
- switch between single-review and specialist-review modes based on risk and scope

### 4. Stronger false-positive control

Added dedicated guidance for:

- confidence thresholds
- reachability checks
- severity control
- duplicate suppression
- pre-existing versus introduced issues

This is one of the most important quality upgrades in the release.

### 5. Structured output contract

Added a formal review output shape so findings can be more consistent and easier to consume in downstream tools.

### 6. Smaller, more maintainable reference layout

The reference set was redesigned to be easier to maintain.

Current focused references:

- `review-framework.md`
- `output-contract.md`
- `false-positive-control.md`
- `security.md`
- `typescript.md`
- `backend-and-integrations.md`
- `verification-and-operations.md`
- `database.md`

### 7. Lighter prompt asset structure

The earlier multi-file reviewer prompt layout was reduced.

What remains:

- `agents/openai.yaml`
- `agents/synthesizer.md`

This keeps the reusable synthesis asset while reducing maintenance overhead.

### 8. Tests and verification

Added automated tests for:

- changed-line parsing
- stack detection
- risk flag derivation
- safe check generation
- review mode selection
- markdown review brief generation
- CLI integration for review context collection
- CLI integration for review brief generation

## Repository Docs

The README was updated to explain:

- what the skill does
- why it is useful
- package structure
- install path for current Codex skill directories
- the two main script entrypoints
- local verification

## Validation Used For This Release

The release was validated with:

```bash
python3 -m unittest discover production-code-quality-review/tests -v
python3 production-code-quality-review/scripts/collect-review-context.py --repo .
python3 production-code-quality-review/scripts/review-entrypoint.py --repo . --format markdown
```

## Breaking Or Notable Changes

- `references/language-specific.md` has been removed
- some earlier granular references were merged into broader, easier-to-maintain guides
- reviewer prompt sprawl was removed in favor of a leaner structure

## Intended Outcome

This release makes `production-code-quality-review` more usable as a real Codex skill artifact:

- easier to install
- easier to understand
- easier to maintain
- more deterministic
- less noisy during review
