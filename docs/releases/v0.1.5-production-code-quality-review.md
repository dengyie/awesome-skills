# v0.1.5: Clean Skill Install Artifacts

## Summary

This patch release tightens the packaging and local update path for `production-code-quality-review`.

It keeps installed skill copies free of Python bytecode caches and source-copy metadata leakage, while preserving the existing review workflow and compact tool surface.

## What Changed

### Install And Update Hygiene

- `install-local-skill.sh` now copies the skill tree with cache exclusions.
- `update-local-skill.sh` uses the same clean-copy behavior when refreshing an installed copy.
- `.skill-source-dir` remains written only as target metadata and is no longer copied from a source tree.
- `__pycache__/` and `*.pyc` artifacts are excluded from installed skill copies.

### Runtime Cleanliness

- Python entrypoints now set `sys.dont_write_bytecode = True` before importing the shared helper library.
- Running an installed entrypoint no longer creates `scripts/__pycache__` inside the installed skill directory.
- `verify-release.sh` exports `PYTHONDONTWRITEBYTECODE=1` so release verification does not dirty the checkout.

### Skill Discovery

- `SKILL.md` now uses a clearer `Use when...` description focused on trigger conditions.

### Regression Coverage

- Added coverage proving install skips Python cache artifacts.
- Added coverage proving installed entrypoints can run without recreating cache directories.
- Extended update-flow coverage to assert the installed copy remains clean.
- Added coverage for the release verifier bytecode-suppression setting.

## Validation

```bash
find production-code-quality-review /Users/mango/.agents/skills/production-code-quality-review \( -name '__pycache__' -o -name '*.pyc' \) -print
```

Result:

- `32` tests passed
- whitespace check passed
- release verification completed successfully
- no Python cache artifacts remained in the repo skill tree or installed skill copy

## Upgrade Notes

Users with an existing local install can refresh it with:

```bash
bash production-code-quality-review/scripts/update-local-skill.sh
```

The canonical Codex user-scope install path remains:

```text
~/.agents/skills/production-code-quality-review
```

Use `INSTALL_LEGACY_CODEX_COPY=1` only when a deliberate legacy copy under `~/.codex/skills` is required.
