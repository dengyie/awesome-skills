# v0.1.4: Review Routing And Scope Controls

## Summary

This release tightens the production review workflow by making review routing explicit and making scope overrides deterministic.

It focuses on helping users and automation understand why a review is routed as low, medium, or high risk, while keeping the skill surface compact.

## What Changed

### Routing Explainability

- Structured context now includes `risk_level` and `review_mode_reason`.
- Markdown and compact briefs now expose the selected risk level.
- API and network-boundary changes are treated as high-risk production surfaces.

### Scope Controls

- `collect-review-context.py` and `review-entrypoint.py` now both support `--base <ref>`.
- Both entrypoints now support `--scope branch` and `--scope working_tree`.
- `branch` scope now excludes uncommitted working-tree files instead of merely changing the output label.

### Documentation

- Added English and Chinese release-note templates.
- Updated examples, quickstart, review workflow, release checklist, and release indexes.
- Synchronized Chinese docs for the new scope override workflow.
