# v0.1.6：Skill Package Contract

## 摘要

这个版本收紧了 `production-code-quality-review` 的 skill 包契约，同时避免给仓库增加不必要的结构噪音。

重点是让 skill 目录更清晰地成为项目核心资产，补上轻量 schema，增强 deterministic review context 收集能力，并保持文档与开发记录边界清楚。

## 具体变化

### 核心 Skill 资产

- 在 `production-code-quality-review/` 内增加 package-level README。
- 在根 README 中更明确地突出 `production-code-quality-review/` 是项目核心。
- 继续把开发记录隔离在 `docs/dev/`，避免污染 skill 本体。

### Schema 契约

- 增加 `review-context.schema.json`，描述生成的 review context 结构。
- 增加 `finding.schema.json`，描述机器可读 finding 结构。
- 增加回归测试，覆盖 schema 契约和核心 skill 包文件存在性。

### Review Context 收集

- `working_tree` scope 现在会合并 branch diff、tracked worktree diff 和 untracked text file ranges。
- base 推断现在会考虑 remote default branch、`develop` 和 `trunk`。
- 本地 review workflow 的 changed-line 覆盖更完整。

### 验证建议

- 改进 `pnpm`、`yarn`、`bun` 和 npm 项目的 package-manager-aware 命令建议。
- 增加 Python 检查识别，包括 `pytest`、`ruff` 和 `mypy`。
- 当 `package.json` scripts 为空时，不再建议必然不存在的 JavaScript 命令。
- Bun 项目存在 test script 时，现在建议 `bun run test`。

## 验证
