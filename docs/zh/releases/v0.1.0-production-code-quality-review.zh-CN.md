# v0.1.0：Production Code Quality Review

## 摘要

这个版本把 `production-code-quality-review` 从一份小型 review SOP，升级为更完整、可测试的 Codex skill 包。

现在这个 skill 组合了：

- 清晰的 review workflow
- 确定性的 repo-context 收集
- changed-line 映射
- 基于技术栈的 reference 路由
- 显式的 false-positive control
- 结构化的 review output contract

## 为什么要做这个版本

原始版本的审查理念是对的，但整体还更像高质量 prompt，而不是可复用的 skill 产品。

这个版本主要补了下面几类缺口：

- review scope 收集不稳定
- 技术栈 guidance 太薄
- 缺少确定性的 changed-line 分析
- 缺少机器可读的 context 输出
- 缺少经过测试的 helper tooling
- 缺少正式 output contract
- 结构过散，不利于长期维护

## 主要变化

### 1. 确定性的 review context 收集

新增一层 Python helper，用来检查 git 仓库并输出结构化 review context。

核心能力包括：

- 推断 base branch，或回退到当前 scope
- 区分 staged、unstaged、untracked 文件
- 收集 changed files
- 将 `git diff --unified=0` 解析为 changed-line ranges
- 通过文件模式检测技术栈信号
- 推导 risk flags
- 推荐安全的验证命令

主要文件：

- `production-code-quality-review/scripts/review_skill_lib.py`
- `production-code-quality-review/scripts/collect-review-context.py`
- `production-code-quality-review/scripts/review-entrypoint.py`

### 2. 面向人的 review brief 输出

新增统一入口，可输出：

- 机器可读 JSON
- markdown review brief

这样不管是在 Codex 内，还是在本地自动化、终端工作流中，都更容易使用。

### 3. 更合理的 review 路由

这个 skill 现在会更有意识地路由审查：

- 先加载核心 review 控制项
- 仅在相关时追加 stack-aware references
- 根据风险和范围在 single-review 与 specialist-review 模式之间切换

### 4. 更强的误报抑制

新增专门 guidance，覆盖：

- confidence threshold
- reachability check
- severity control
- duplicate suppression
- pre-existing 与 introduced issue 的区分

这是这个版本最重要的质量升级之一。

### 5. 结构化输出契约

新增正式的 review output shape，让 findings 更一致，也更便于被下游工具消费。

### 6. 更小、更易维护的 references 结构

references 经过重构，维护成本更低。

当前保留的核心 references：

- `review-framework.md`
- `output-contract.md`
- `false-positive-control.md`
- `security.md`
- `typescript.md`
- `backend-and-integrations.md`
- `verification-and-operations.md`
- `database.md`

### 7. 更轻的 prompt 资产结构

之前偏分散的 reviewer prompt 布局被压缩了。

保留内容：

- `agents/openai.yaml`
- `agents/synthesizer.md`

这样既保留了可复用的 synthesis 资产，也降低了维护成本。

### 8. 测试与验证

新增自动化测试，覆盖：

- changed-line parsing
- stack detection
- risk flag derivation
- safe check generation
- review mode selection
- markdown review brief generation
- review context collection 的 CLI 集成
- review brief generation 的 CLI 集成

## 仓库文档更新

README 也同步更新，明确说明了：

- skill 的定位
- skill 的使用价值
- 包结构
- 当前 Codex skill 目录的安装路径
- 两个主要脚本入口
- 本地验证方式

## 本版本采用的验证方式

```bash
python3 -m unittest discover production-code-quality-review/tests -v
python3 production-code-quality-review/scripts/collect-review-context.py --repo .
python3 production-code-quality-review/scripts/review-entrypoint.py --repo . --format markdown
```

## Breaking 或 Notable Changes

- `references/language-specific.md` 已删除
- 一些更细碎的 reference 已合并为更易维护的指南
- reviewer prompt 的分散结构被移除，改为更精简的布局

## 目标结果

这个版本让 `production-code-quality-review` 更像一个真正可用的 Codex skill 产物：

- 更容易安装
- 更容易理解
- 更容易维护
- 更确定性
- review 过程更少噪音
