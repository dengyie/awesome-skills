# v0.1.3：Installed Copy Update Recovery

## 摘要

这个版本修复了本地 skill 维护流程里优先级最高的一个问题。

它聚焦于：

- 让已安装的 skill 副本记住自己的来源 checkout
- 让 `update-local-skill.sh` 从已安装路径执行时也能安全刷新
- 把新的安全更新行为补充到文档中

## 具体变化

### 更新流程

已安装副本现在会把来源 checkout 路径写入 `.skill-source-dir`。

`update-local-skill.sh` 会读取这份元数据，并从记录的来源 checkout 刷新内容，因此即使从已安装副本路径执行，也不会再把自己的源目录删掉。

### 可靠性

新增针对已安装副本更新路径的回归测试。

### 文档

README、quickstart、golden path、changelog 和 release 文档都已更新为新的记录来源刷新模型。

## 验证

验证命令：

```bash
python3 -m unittest discover production-code-quality-review/tests -v
bash production-code-quality-review/scripts/install-local-skill.sh
bash /Users/mango/.agents/skills/production-code-quality-review/scripts/update-local-skill.sh
```

## 备注

这是一个建立在 `v0.1.2` 之上的聚焦补丁版本。review 模型和 helper 入口保持不变，但维护路径已经更安全、更可预期。
