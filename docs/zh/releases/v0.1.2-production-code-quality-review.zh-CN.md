# v0.1.2：Installation And Packaging Polish

## 摘要

这个版本主要做最后一层产品化收口，重点放在本地安装体验和 GitHub 面向用户的包装质量上。

它聚焦于：

- 让辅助脚本对齐当前推荐的 `~/.agents/skills` 安装路径
- 将旧版 `~/.codex/skills` 同步改为显式开关
- 提升 README 在 GitHub 首页上的可发现性
- 移除被跟踪的缓存产物，让 skill 包更干净

## 具体变化

### 安装

`install-local-skill.sh` 和 `update-local-skill.sh` 现在默认安装到 `~/.agents/skills`。

只有在设置 `INSTALL_LEGACY_CODEX_COPY=1` 时，脚本才会同步旧版 `~/.codex/skills`。

同时，安装副本会记录来源 checkout，`update-local-skill.sh` 可以利用这份元数据在已安装路径中安全回源刷新。

### 文档

仓库首页现在补充说明了：

- 基于 helper 的安装方式
- `compact` review brief 模式
- 便于快速导航的仓库结构

usage 文档也直接解释了安装辅助脚本的行为。

### Packaging

从 skill 包里移除了被跟踪的 `__pycache__` 产物，让仓库发布的是更干净的 source-only 内容。

## 验证

```bash
python3 -m unittest discover production-code-quality-review/tests -v
bash production-code-quality-review/scripts/install-local-skill.sh
bash production-code-quality-review/scripts/update-local-skill.sh
python3 production-code-quality-review/scripts/review-entrypoint.py --repo . --format compact
```

## 备注

这是一个 polish release。review model、确定性的 context collection 和 reference-routing 逻辑都保持不变。
