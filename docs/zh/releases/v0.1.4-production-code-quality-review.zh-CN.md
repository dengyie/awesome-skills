# v0.1.4：Review Routing And Scope Controls

## 摘要

这个版本把生产级 review 工作流继续收紧：review routing 更可解释，scope override 更确定。

重点是让用户和自动化流程都能看懂为什么一次 review 被判定为低、中或高风险，同时保持 skill 表面足够轻。

## 具体变化

### 路由可解释性

- 结构化上下文新增 `risk_level` 和 `review_mode_reason`。
- Markdown 与 compact 简报都会展示风险等级。
- API 与 network boundary 改动会被视为高风险生产面。

### Scope 控制

- `collect-review-context.py` 和 `review-entrypoint.py` 都支持 `--base <ref>`。
- 两个入口都支持 `--scope branch` 和 `--scope working_tree`。
- `branch` scope 现在会真正排除未提交工作区文件，而不只是改变输出标签。

### 文档

- 新增英文和中文 release note 模板。
- 更新 examples、quickstart、review workflow、release checklist 和 release index。
- 中文文档已同步新的 scope override 工作流。

## 备注

这个版本有意保持紧凑。它没有增加新的 reviewer prompt 分裂，而是强化两个主要入口，并补齐围绕它们的发布文档。
