# v0.1.5：Clean Skill Install Artifacts

## 摘要

这个补丁版本收紧了 `production-code-quality-review` 的打包、本地安装和更新路径。

重点是让已安装的 skill 副本不再残留 Python bytecode cache，也避免源目录里的 metadata 被误复制到安装产物中，同时保持现有 review 工作流和工具表面不膨胀。

## 具体变化

### 安装与更新清洁度

- `install-local-skill.sh` 使用带排除规则的复制流程。
- `update-local-skill.sh` 刷新已安装副本时使用同样的 clean-copy 行为。
- `.skill-source-dir` 只作为目标目录 metadata 写入，不再从源目录复制。
- 已安装 skill 副本会排除 `__pycache__/` 和 `*.pyc`。

### 运行时清洁度

- Python 入口脚本在导入共享 helper 前设置 `sys.dont_write_bytecode = True`。
- 运行已安装入口脚本时，不再在安装目录生成 `scripts/__pycache__`。
- `verify-release.sh` 导出 `PYTHONDONTWRITEBYTECODE=1`，避免发布验证弄脏 checkout。

### Skill 发现

- `SKILL.md` 的 description 调整为更清晰的 `Use when...` 触发条件描述。

### 回归测试

- 增加测试，证明安装流程会跳过 Python cache artifacts。
- 增加测试，证明运行已安装入口不会重新创建 cache 目录。
- 扩展更新流程测试，确认已安装副本保持干净。
- 增加发布验证脚本 bytecode 抑制设置的覆盖。

## 验证

验证命令：

```bash
PYTHONDONTWRITEBYTECODE=1 python3 -m unittest discover production-code-quality-review/tests -v
git diff --check
bash production-code-quality-review/scripts/verify-release.sh
find production-code-quality-review /Users/mango/.agents/skills/production-code-quality-review \( -name '__pycache__' -o -name '*.pyc' \) -print
```

结果：

- `32` 个测试通过
- whitespace 检查通过
- release verification 成功完成
- 仓库 skill tree 和本地安装副本都没有 Python cache artifact 残留

## 升级说明

已有本地安装的用户可以这样刷新：

```bash
bash production-code-quality-review/scripts/update-local-skill.sh
```

当前推荐的 Codex user-scope 安装路径仍然是：

```text
~/.agents/skills/production-code-quality-review
```

只有明确需要旧版兼容副本时，才使用 `INSTALL_LEGACY_CODEX_COPY=1` 同步到 `~/.codex/skills`。
