# V27 Zero-To-Website Visual Delivery Closure Plan

## Problem

The V22-V26 hardening passes added visual direction selection, fidelity audits, verifiable evidence, interactive implementation rules, and design rounds. Real use still showed a gap: agents can obey the words while drifting into loose visual implementation, weak self-comparison, late asset decisions, or overconfident delivery language.

## Goal

Make visually driven website delivery harder to overclaim by adding explicit gates for user-selected visual authority, reference-to-DOM decomposition, component-slot asset strategy, top visible difference review, visual usability, user-feedback status updates, and final visual pass reporting.

## Scope

- Strengthen `zero-to-website-design/SKILL.md` without bloating it.
- Move detailed operational rules into existing references.
- Extend templates so downstream projects naturally capture the new evidence.
- Add regression coverage for the new contract.
- Update public usage docs and project memory.

## Non-Goals

- No new visual-diff automation script.
- No new forward-test thread in this pass.
- No rewrite of the existing design-round or fidelity-loop architecture.

## Acceptance

- The skill blocks `Visual Delivery Ready` when visual work lacks user-selected concrete image authority.
- Binding references require a Reference-To-DOM Map before broad implementation.
- Asset generation requires component-slot prompt records.
- Final visual QA names the top 3 visible differences before each next edit pass.
- Final reporting includes URL, binding reference, latest screenshots, runtime guards, verification, gaps, and status.
